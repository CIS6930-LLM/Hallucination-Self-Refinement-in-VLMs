# Source Code Layout

- `data_pipeline/`: Scripts for ingestion, preprocessing, and dataset generation.
- `rationale_module/`: Components that predict rationales or visual groundings.
- `self_refinement/`: Modules implementing the verification loop, contrastive objectives, and revision policies.
- `evaluation/`: Metrics, benchmarking pipelines, and reporting utilities.
- `utils/`: Shared helpers (configuration loading, logging, etc.).
